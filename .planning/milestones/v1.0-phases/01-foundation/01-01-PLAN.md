<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser storage (read/write) | App writes to IndexedDB via Dexie — data lives entirely on the local device, no network surface |
| No network surface | Phase 1 makes zero fetch/XHR calls. Only external connection is the Google Fonts CDN @import in CSS (loaded by the browser, not by app JS) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | src/db/db.ts — AppConfig table | accept | No sensitive data in Phase 1. The 'pinHash' key is defined in the schema but Phase 1 writes only 'onboardingComplete'. PIN hashing is a Phase 5 concern — NEVER store PIN as plaintext (enforce in Phase 5 with bcrypt/SubtleCrypto). |
| T-01-02 | Tampering | IndexedDB MathTutorDB | accept | Any user with device access can open DevTools and edit IndexedDB. This is accepted: the app is single-user, local-only, no backend to corrupt. Data integrity is a usability concern, not a security threat for this threat model. |
| T-01-03 | Denial of Service | Safari 7-day IndexedDB eviction | mitigate | Safari evicts IndexedDB data for origins not visited in 7 days (iOS 17+). Mitigation: document in CLAUDE.md that Phase 6 (PWA) must evaluate whether a Service Worker visit counts as user interaction to reset the eviction timer. Phase 5 parent dashboard should surface a data-loss warning per v2 requirement PAR-V2-03. |
| T-01-04 | XSS | React JSX rendering | accept | React's JSX escapes all string values by default. Phase 1 renders no user-supplied input — all content is hardcoded strings. dangerouslySetInnerHTML is NEVER used. Threat is accepted at negligible residual risk. |
| T-01-05 | Information Disclosure | Google Fonts CDN @import | accept | The browser makes a request to fonts.googleapis.com at load time, which reveals the user's IP to Google. Accepted for Phase 1 (standard web practice). Phase 6 Service Worker research should evaluate self-hosting Nunito for full offline + privacy. |

**No authentication surface in Phase 1.** PIN gate (PLAT-03 / PAR-01) is deferred to Phase 5.
</threat_model>
