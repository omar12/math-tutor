## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser storage (read/write) | HomeScreen writes to IndexedDB on mount — same local-only surface as Plan 01 |
| No network surface | All UI content is hardcoded strings. No fetch calls. Google Fonts CDN loaded via CSS @import by the browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | src/screens/ParentScreen.tsx | accept | Parent screen has no PIN gate in Phase 1. This is intentional — stub only. PIN gate implemented in Phase 5 (PAR-01). Document clearly in stub screen that PIN gate is deferred. |
| T-02-02 | Spoofing | src/screens/HomeScreen.tsx — useNavigate | accept | Navigation is client-side only; no server authentication. Any user with device access can navigate to /parent. Accepted for Phase 1 stub; PIN gate is Phase 5 mitigation. |
| T-02-03 | Tampering | db.appConfig.put in HomeScreen | accept | Overwrites onboardingComplete on every mount — idempotent, low-value key. No sensitive data written in Phase 1. pinHash key defined in schema but not written until Phase 5. |
| T-02-04 | XSS | All screen components — hardcoded strings | accept | All rendered content is hardcoded string literals in JSX. No user input, no innerHTML, no dangerouslySetInnerHTML. React escapes all JSX string expressions by default. Residual risk is negligible. |
</threat_model>
