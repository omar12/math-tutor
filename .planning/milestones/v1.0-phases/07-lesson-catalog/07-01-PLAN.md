## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Curriculum data → HomeScreen render | lessons array is static build-time data; no user input crosses this boundary |
| IndexedDB (Dexie) → React state | useLiveQuery reads sessions and topicProgress written by the practice engine in earlier phases |
| User tap → navigate() | Lesson ID comes from curriculum constant, not user-supplied string |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | navigate('/lesson/' + lesson.id) | accept | lesson.id is sourced from the static curriculum constant, never from user input or DB string; no sanitization needed |
| T-07-02 | Information Disclosure | db.sessions read in HomeScreen | accept | sessions contain lessonId and score data — no PII, no credentials; local device storage only, no network boundary |
| T-07-03 | Denial of Service | useLiveQuery over 27 lessons + sessions | accept | 27 lessons and bounded sessions array; Map/Set derivation is O(n) with n < 1000 realistically; no performance risk |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies introduced in this plan; all libraries already installed in package.json |
