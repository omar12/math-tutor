<objective>
Install Howler.js and create the useLessonAudio hook that manages per-step Howl instances. Also wire the Howler module mock into the test setup so subsequent plans can test audio-dependent components.

This plan establishes the audio infrastructure that Plan 03 (LessonScreen) depends on. The hook's design is the core of LESS-02 and LESS-04: one Howl per step created at mount, synchronous play on unlock tap (Plan 03 calls it), and silent fallback via onloaderror dispatching onStepEnded (D-06, Pitfall 5).

    Implementation requirements (in order):

    1. Import: `import { Howl } from 'howler'` and `import { useMemo, useEffect } from 'react'`

       - Prevents ghost audio and memory leak (Pitfall 3)
       - deps array: [howls]

    5. Return howls array

    Do not add html5: true. Do not add any console.error or log statements (D-06: silent failure). Do not add any state inside the hook.
  </action>
