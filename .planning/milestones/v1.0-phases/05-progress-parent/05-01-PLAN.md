<objective>
Instrument PracticeScreen to record a completed session to IndexedDB when the celebration screen is reached. After writing the Session, immediately recalculate and upsert the TopicProgress record for that topic by aggregating all sessions. Both writes are wrapped in a single Dexie transaction so they succeed or fail atomically. A useRef guard prevents duplicate writes when React re-renders during celebration.

Purpose: Provides the accuracy data that Phase 5's adaptive ordering (Plan 03) and parent dashboard (Plan 03) will consume. Without this write path, those features have no data.

Output: PracticeScreen emits one Session + one TopicProgress upsert per completed practice session. Tests confirm the write happens on celebration, does not happen on mid-session quit, and does not duplicate on re-render.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PracticeScreen → db.sessions | Session write triggered by internal React state (celebration phase), not user-controlled input — low risk |
| PracticeScreen → db.topicProgress | Aggregate upsert driven by db.sessions query result — data is internal only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-P01 | Tampering | practiceReducer — correctCount | mitigate | correctCount is internal React state, not user-provided; only increments on CORRECT_ANSWER dispatch which is triggered by checking against curriculum answer |
| T-5-P02 | Information Disclosure | db.sessions — written plaintext | accept | Session data (lessonId, topic, grade, score) is non-sensitive educational telemetry; no PII stored; local device only |
| T-5-P03 | Denial of Service | db.sessions.add() throws | accept | Write is non-blocking (.catch logs error); celebration still renders if write fails — child experience unaffected |
| T-5-P04 | Tampering | useEffect re-fire during celebration → duplicate session | mitigate | hasRecorded useRef guard prevents second write; ref is reset in effect cleanup on unmount |
| T-5-P05 | Tampering | Partial write (session written, progress not updated) | mitigate | db.transaction('rw', [db.sessions, db.topicProgress], ...) wraps both writes — Dexie rolls back on any failure |
| T-5-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase — no install surface |
