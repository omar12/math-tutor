<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ParentScreen → topicProgress table | Read-only display; no user input crosses this boundary |
| HomeScreen → topicProgress table | Read-only for adaptive sort; user cannot influence accuracy data directly from HomeScreen |
| HomeScreen lock icon → /pin | Navigation only — no data transfer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-06 | Elevation of Privilege | HomeScreen "Parent" button navigating directly to /parent | mitigate | Remove the existing navigate('/parent') button entirely (per D-01); replace with lock icon navigating to /pin; the old direct route is eliminated from UI |
| T-5-07 | Information Disclosure | ParentScreen showing accuracy data without PIN | accept | ParentScreen is only reachable via /parent, which is only navigated to by PinScreen after successful verifyPin(); HomeScreen no longer links directly to /parent; v1 local-only threat model accepts this (no network surface) |
| T-5-08 | Tampering | sortedLessons mutating the lessons array | mitigate | Use [...lessons].sort(...) spread-before-sort pattern; curriculum order immutable for subsequent renders; covered by dedicated mutation-detection test |
| T-5-09 | Denial of Service | useLiveQuery undefined crash in ParentScreen or HomeScreen | mitigate | Explicit undefined guard before any .map() or .find() call; loading state rendered while query resolves |
| T-5-10 | Tampering | Corrupted accuracy value (> 1.0 or < 0) causing overflowing fill bar | mitigate | ProgressBar clamps accuracy with Math.min(1, Math.max(0, accuracy)) before computing pct — bar width cannot exceed 100% or go below 0% |
| T-5-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase |
</threat_model>
