## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Digit key inner padding, label gaps |
| md | 16px | Card internal padding, question-to-answer gap |
| lg | 24px | Section padding, card-to-card gap |
| xl | 32px | Screen vertical padding top/bottom |
| 2xl | 48px | Major vertical breaks (question block to input block) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets floored at 44px per Apple HIG (CLAUDE.md). Digit grid keys: minimum 64×64px to accommodate fat-finger accuracy on ages 6–9.
- Answer option cards: minimum 64px height (2× the 44px floor) for comfortable tap on large equations.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 18px (text-lg) | 400 (regular) | 1.5 |
| Label | 16px (text-base) | 700 (bold) | 1.4 |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 |
| Display (equation / answer) | 40px (text-4xl) | 700 (bold) | 1.1 |

Notes:
- Problem question text uses Heading (24px / 700) — large enough for grades 1–3 readers.
- Equation expressions inside answer option cards use Display (40px / 700) — matches StepCard equation size from Phase 3.
- Digit display box (composed answer) uses Display (40px / 700) — number must read clearly before submission.
- Digit grid keys use Heading (24px / 700) — high contrast, easily read on 64px targets.
- Feedback / encouragement phrases use Body (18px / 400) — subordinate to the problem, not alarming.
- "Check Answer" button label uses Label (16px / 700).

Source: Phase 1 D-11 (Nunito globally), Phase 3 StepCard.tsx (text-xl / text-4xl pattern), Phase 1 D-12 (kids 6–9 legibility requirement).

## Component Inventory

### PracticeScreen (replaces stub)
- Full-screen container: `style={{ height: '100dvh', paddingBottom: 'env(safe-area-inset-bottom)' }}` + `bg-surface`
- Layout: flex column, top-to-bottom: [progress indicator] → [problem question] → [feedback slot] → [input widget]
- Progress indicator: dot row (filled dots = answered, outline dot = current, empty = remaining). Dots are decorative only — not tappable. 8px dots, 8px gap.

### MultipleChoiceWidget
- 4 answer option cards in a 2×2 grid on portrait, 1×4 column on landscape (CSS grid responsive)
- Each card: `bg-white rounded-2xl shadow-sm` + 64px min-height + full width in its grid cell
- Card content: equation/number centered, Display size (40px / 700)
- Tap immediately submits (D-04) — no selected state persists
- Wrong-answer card: 300ms orange border pulse (`border-2 border-primary`), then resets to neutral
- Correct-answer card (on reveal, D-09): `border-2 border-success bg-success/10` held for 1.5s before auto-advance

### DigitGridWidget
- Digit display box above grid: `bg-white rounded-2xl shadow-sm` + 64px height + Display size (40px / 700)
  - Empty state: single underscore character `_` in `text-on-surface/30`
  - Filled state: composed digits left-aligned in display box
- 3×4 grid of digit keys (0–9 + backspace + Check Answer): each key 64×64px minimum, `bg-white rounded-xl shadow-sm`
  - Digit keys (0–9): Heading size (24px / 700), `text-on-surface`
  - Backspace key: inline SVG backspace icon, `stroke-primary` (same orange as replay icon)
  - Check Answer key: `bg-accent text-white` Label size (16px / 700) — only enabled when display box is non-empty; disabled state: `opacity-40`
- Grid layout: `grid grid-cols-3 gap-4`
- Key layout row order: [7][8][9] / [4][5][6] / [1][2][3] / [backspace][0][Check Answer]

### FeedbackSlot
- Fixed slot below problem question — always occupies vertical space to prevent layout shift (D-08)
- Empty (no attempt yet): invisible, height reserved (min-h-[48px])
- Warm encouragement (after 1st or 2nd wrong): `text-primary` Body size (18px / 400), centered
- Hint (after 2nd wrong, D-08): same slot, same style, text changes to hint content
- No icon, no border — text only, keeps visual calm

### Encouragement Phrases (rotating, FEED-03)
Warm, non-punishing. Remy the Fox voice. Rotate sequentially (not randomly) to avoid repetition within a session:
1. "Nice try! Give it another go."
2. "Almost! You can do it."
3. "Keep going — you're doing great!"
4. "Not quite, but don't give up!"
5. "Hmm, try again!"

Hint slot (after 2nd wrong) uses problem-specific hint from curriculum data. Fallback hint if none present: "Think about what number makes the equation balance."

### ConfettiScreen (reused, extended)
- Existing component in `src/components/ConfettiScreen.tsx`
- Add prop: `buttonLabel?: string` (default `'Start Practice'` — backwards compatible with Phase 3)
- Phase 4 usage: `buttonLabel="Back to Home"` + `onStartPractice={() => navigate('/')}`
- No visual changes to the component — only the button label changes

## Interaction Contract

### Multiple-Choice Flow
1. Problem question displayed, 4 cards rendered (choices shuffled once at render, D-08 from Phase 2).
2. Child taps a card → immediate submission (D-04).
3. If correct: green flash on card (200ms) → advance to next problem.
4. If wrong (attempt 1): orange border pulse on tapped card, encouragement phrase appears in FeedbackSlot.
5. If wrong (attempt 2): orange border pulse, hint replaces encouragement phrase in FeedbackSlot.
6. If wrong (attempt 3, D-09): correct card gets `border-success bg-success/10`, held 1.5s → auto-advance (no tap needed).
7. Attempt counter resets per problem.

### Digit-Grid Flow
1. Problem question displayed, digit display box shows `_`, Check Answer disabled.
2. Child taps digit keys → digits compose in display box. Check Answer enables after first digit.
3. Backspace removes last digit. If display box becomes empty, Check Answer disables again.
4. Child taps Check Answer → submission.
5. If correct: display box flashes green border (200ms) → advance.
6. If wrong (attempt 1): display box flashes orange border (300ms), encouragement in FeedbackSlot. Display box clears.
7. If wrong (attempt 2): same + hint replaces encouragement. Display box clears.
8. If wrong (attempt 3, D-09): correct number appears in display box in `text-success` color, held 1.5s → auto-advance.

### Session End
- Last problem answered (correctly or revealed) → ConfettiScreen with `buttonLabel="Back to Home"`.
- No auto-advance to celebration — transition is instant after the 1.5s reveal pause on final problem (if revealed) or after the correct-flash on the final problem.

### Animation Budget
- Wrong-answer card pulse: CSS `@keyframes`, 1 cycle, 300ms. No Motion/Framer.
- Correct-answer flash: CSS opacity transition, 200ms. No Motion/Framer.
- Correct-reveal hold (1.5s): `setTimeout` + state transition. No animation library.
- Confetti: existing `confetti-fall` keyframe in `src/index.css`. No new animation code.
- No Lottie for any per-problem moment (CONTEXT.md Claude's Discretion constraint).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (digit-grid submit) | "Check Answer" |
| Session-end button | "Back to Home" |
| Empty display box placeholder | `_` (single underscore, visually understated) |
| Progress dots aria-label | "Problem {N} of {M}" (on the container, for screen readers) |
| Backspace key aria-label | "Delete last digit" |
| Check Answer button aria-label | "Check my answer" |
| Correct-reveal status (screen reader) | "The answer is {N}. Moving to the next problem." (aria-live region) |
| Wrong attempt 1 encouragement (example) | "Nice try! Give it another go." (rotates through 5 phrases above) |
| Hint (attempt 2) | Problem-specific hint from curriculum; fallback: "Think about what number makes the equation balance." |
| No empty state | Not applicable — practice always has ≥1 problem (curriculum guarantees 5 per lesson) |
| Error state (problem load failure) | Not surfaced to child — if problem pool is empty, navigate directly to ConfettiScreen (silent fallback, consistent with Phase 3 audio fallback pattern D-06) |

No destructive actions in this phase.

## Accessibility

- All interactive elements meet 44px minimum (globally enforced by `src/index.css` button rule).
- Digit grid keys use 64px — exceeds minimum for ages 6–9 fat-finger tolerance.
- `touch-action: manipulation` on all tappable elements (globally enforced).
- `user-select: none` on problem text (prevents accidental text selection on drag).
- `aria-live="polite"` region for feedback slot — screen reader announces encouragement and hint changes.
- `aria-live="assertive"` region for correct-reveal message — announces correct answer before auto-advance.
- ConfettiScreen confetti pieces remain `aria-hidden="true"` (already implemented).
- Answer option cards: `aria-label="{choice value}"` (e.g., `aria-label="7"`).
