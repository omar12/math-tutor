# Requirements: Math Tutor — Kids Edition

**Defined:** 2026-05-12  
**Core Value:** A kid sits down, does a lesson, practices, and the parent can see exactly what their child understands and what they don't — without any accounts or setup friction.

## v1 Requirements

### Platform

- [x] **PLAT-01**: App runs in Safari on iPad, fully touch-optimized with no mouse assumptions — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)
- [x] **PLAT-02**: App uses `100dvh` and `env(safe-area-inset-bottom)` to handle iPad viewport correctly — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)
- [x] **PLAT-03**: Single profile per device — child can open app and start immediately with no login or setup — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)
- [x] **PLAT-04**: All progress data persists locally via IndexedDB across sessions — ✅ v1.0 (db.test.ts: 2 persistence tests pass)

### Curriculum

- [x] **CURR-01**: App covers grade 1–3 addition and subtraction content aligned to Common Core standards — ✅ v1.0 (VERIFICATION.md passed 12/12)
- [x] **CURR-02**: App includes grade 1–3 word problems with narration reading the problem aloud — ✅ v1.0 (VERIFICATION.md passed 12/12)
- [x] **CURR-03**: Curriculum is data-driven (static JSON) so adding content requires no code changes — ✅ v1.0 (VERIFICATION.md passed 12/12)

### Lesson

- [x] **LESS-01**: Each lesson begins with a guided worked example that walks through a sample problem step by step with narration — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)
- [x] **LESS-02**: All lesson instructions and problem text are narrated aloud (pre-recorded audio) — ✅ v1.0 (useLessonAudio.test.ts: 5 tests pass; audio stubs in place)
- [x] **LESS-03**: Kid can tap any narrated segment to replay it — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)
- [x] **LESS-04**: Audio playback is unlocked via the first deliberate user tap (satisfies iOS Safari gesture requirement) — ✅ v1.0 (SUMMARY evidence; no VERIFICATION.md)

### Practice

- [x] **PRAC-01**: Kid can answer practice problems via multiple choice — 4 large-touch-target options, no keyboard — ✅ v1.0 (PracticeScreen.test.tsx: 3 named tests pass; no VERIFICATION.md)
- [x] **PRAC-02**: Kid can answer fill-in-the-blank problems via a tappable digit grid (0–9), no software keyboard — ✅ v1.0 (PracticeScreen.test.tsx: 6 named tests pass; no VERIFICATION.md)
- [x] **PRAC-03**: Practice problems follow immediately after each lesson's guided example — ✅ v1.0 (route wiring confirmed; no VERIFICATION.md)

### Feedback

- [x] **FEED-01**: After 2 wrong attempts on a problem, app shows a hint — ✅ v1.0 (PracticeScreen.test.tsx: named test passes)
- [x] **FEED-02**: After 3 wrong attempts, app reveals the correct answer and advances — no infinite retry loop — ✅ v1.0 (PracticeScreen.test.tsx: named test passes)
- [x] **FEED-03**: Encouragement phrases on wrong answers rotate and maintain a warm, non-punishing tone — ✅ v1.0 (PracticeScreen.test.tsx: named test passes)
- [x] **FEED-04**: Session end triggers a completion celebration screen with visual reward — ✅ v1.0 (PracticeScreen.test.tsx: ConfettiScreen test passes)

### Progress

- [x] **PROG-01**: App tracks per-topic accuracy (addition, subtraction, word problems) across sessions — ✅ v1.0 (VERIFICATION.md passed 11/11)
- [x] **PROG-02**: Practice sessions use error-adaptive repetition — topics with lower accuracy appear more frequently — ✅ v1.0 (VERIFICATION.md passed 11/11)

### Parent Section

- [x] **PAR-01**: Parent section is accessible from the main screen behind a 4-digit PIN — ✅ v1.0 (VERIFICATION.md passed 11/11)
- [x] **PAR-02**: Parent dashboard shows accuracy per topic (addition, subtraction, word problems) — ✅ v1.0 (VERIFICATION.md passed 11/11)

### PWA

- [x] **PWA-01**: App can be installed via "Add to Home Screen" on iPad (PWA with Web App Manifest) — ✅ v1.0 (pwa.test.ts: 6 tests pass; iPad verify deferred)
- [x] **PWA-02**: App works fully offline after first load via Service Worker — ✅ v1.0 (Workbox generateSW wired; iPad offline verify deferred)
