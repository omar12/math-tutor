## Blockers

### BLOCKER-01: PROG-02 Regression — `sortedLessons` Dead Code

**Requirement:** PROG-02 — Topics with lower accuracy appear more frequently in practice

`sortedLessons()` is exported from `HomeScreen.tsx:15` and tested in isolation. Phase 5 verified it as SATISFIED. However, Phase 7 rewrote HomeScreen with a lesson catalog that uses a static sort:

```tsx
// HomeScreen.tsx:119 — what's actually running
const topicLessons = lessons
  .filter(l => l.topic === key)
  .sort((a, b) => a.grade !== b.grade ? a.grade - b.grade : a.order - b.order)
```

`topicProgress` data is fetched and used for completion rings, accuracy dots, and the weakest-topic badge — but never used to reorder lessons. Kids always see lessons in fixed curriculum order regardless of their struggle areas.

**Fix:** Call `sortedLessons(lessons, topicProgress)` in HomeScreen catalog section, replacing the static `.sort()`.

---

### BLOCKER-02: PAR-01 — `/parent` Route Unguarded

**Requirement:** PAR-01 — Parent section accessible from the main screen behind a 4-digit PIN

`App.tsx:16` registers `<Route path="/parent" element={<ParentScreen />} />` as an open route. `ParentScreen.tsx` has no auth check, no redirect, and no sessionStorage/context check. Direct URL navigation to `/parent` bypasses PIN.

The PIN *flow* itself is correctly implemented: PinScreen, hashPin, storePinHash, verifyPin are all working. The gap is that the destination is unprotected.

**Fix:** Add a `ProtectedRoute` wrapper that checks a sessionStorage flag set on PIN verification, or redirect unauthenticated users from `/parent` to `/pin`.

---

### BLOCKER-03: Missing Audio Files — LESS-02, LESS-03 Broken

**Requirements:** LESS-02 (narrated instructions), LESS-03 (tap to replay), LESS-04 (iOS audio unlock), CURR-02 (word problems narrated aloud)

`public/audio/` directory does not exist. All 108 `narrationAudio` paths in `curriculum.json` (e.g. `/audio/lessons/addition-grade1-01/step-1.mp3`) return 404. The `useLessonAudio` hook has correct `onloaderror → onStepEnded` fallback so the app advances silently without crashing, but:
- LESS-02: No narration plays
- LESS-03: Nothing to replay
- LESS-04: Audio context unlock is architecturally correct but unverifiable

The lesson visual flow is fully functional — only the audio is missing.

**Fix options:**
1. Record/source 108 MP3 files matching the curriculum path convention
2. Use Web Speech API TTS as fallback until recordings exist
3. Generate placeholder audio files (text-to-speech synthesis at build time)
