<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → digit buttons | User taps 0–9; only these values enter the digits buffer |
| digits buffer → hashPin() | Raw PIN string crosses into crypto layer — must hash before any DB write |
| hashPin() output → storePinHash() | 64-char hex digest stored in appConfig table — never raw digits |
| /pin → /parent | Navigation only happens after verifyPin() returns true or storePinHash() succeeds |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Information Disclosure | PinScreen state (digits, firstEntry) | mitigate | Raw digits live only in React useState during entry; never logged, never written to DB; component unmounts after navigation, clearing state |
| T-5-02 | Elevation of Privilege | Direct navigation to /parent bypassing /pin | mitigate | App.tsx routes do not guard /parent at the router level (v1 local-only accepted per PROJECT.md CR-01 note); PinScreen is the only UI path to /parent in the app; HomeScreen lock icon navigates to /pin (Plan 03) |
| T-5-03 | Elevation of Privilege | PIN brute force | accept | No lockout per D-04 (v1 decision, physical device access required, parent controls device) |
| T-5-04 | Tampering | storePinHash called with raw PIN | mitigate | storePinHash() throws at runtime if argument is not 64-char hex (verified in db.ts); always call hashPin() first |
| T-5-05 | Tampering | Non-digit characters entering digits buffer | mitigate | No text input in PinScreen; only digit buttons (0–9) can append to digits buffer; handleDigit silently ignores calls if digits.length >= 4 |
| T-5-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase |
</threat_model>
