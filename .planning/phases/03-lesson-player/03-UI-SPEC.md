## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gap between replay icon and step text |
| sm | 8px | Inline padding inside equation display |
| md | 16px | Horizontal screen padding (px-4 / px-6 Tailwind) |
| lg | 24px | Gap between StepCard elements (gap-6 Tailwind) |
| xl | 32px | Vertical gap between step text and equation block |
| 2xl | 48px | Top/bottom padding on unlock screen content area |
| 3xl | 64px | "Start Practice" button min-height |

Exceptions:
- All interactive elements (replay icon, Start Practice button, Next indicator): minimum 44×44px touch target — enforced by global CSS in `src/index.css` (`button { min-height: 44px; min-width: 44px }`)
- Unlock screen: entire screen is the tap target — `height: 100dvh`, `width: 100%`
- Dot progress indicators: 12px diameter with 8px gap between dots (below 4px grid but intentionally small to fit without clutter)

Source: CLAUDE.md (44px touch targets), `src/index.css` (global button rule), 03-CONTEXT.md D-07

| Dominant (60%) | `#FFF8F0` (`--color-surface`) | Full-screen backgrounds: unlock screen, step screen, celebration screen |

**Step Screen** (state: `playing` | `between-steps`)

Focal point: equation block (40px bold, centered) when `LessonStep.equation` is present; step instruction text (20px regular) when no equation

```
┌────────────────────────────────────────┐  ← 100dvh
│  [● ● ○ ○ ○]  Step N of M             │  ← dot progress, top-center, 24px from top

│  ┌──────────────────────────────────┐  │
│  │  [Step text — text-xl / 1.5lh]  │  │  ← white card, rounded-2xl, shadow-sm
│  │                                  │  │
│  │  [Equation — text-4xl bold]      │  │  ← FOCAL POINT (when present); if LessonStep.equation present
│  │                                  │  │
│  │                         [↺ icon] │  │  ← replay icon, bottom-right of card, 44×44px
│  └──────────────────────────────────┘  │
│                                        │
│         [→ Next]  (visible only in     │  ← between-steps state; orange #FF6B35
│                    between-steps)      │    min 64×64px, centered, rounded-full
└────────────────────────────────────────┘
```
- Card: white `bg-white rounded-2xl shadow-sm p-8`
- Card occupies center 80% of screen width (`w-4/5 max-w-lg`)
- Replay icon: circular arrow SVG, `#FF6B35`, 44×44px minimum touch target
- Next indicator: appears only when `phase === 'between-steps'`; hidden during `playing`

## Interaction Contract

### Touch Targets

| Element | Minimum Size | Implementation |
|---------|-------------|----------------|
| Unlock screen (entire screen) | 100dvh × 100% | `onClick` on root div |
| Replay icon | 44×44px | `min-h-[44px] min-w-[44px]` + `style={{ touchAction: 'manipulation' }}` |
| Next indicator | 64×64px | `min-h-[64px] min-w-[64px]` — oversized intentionally for kids |
| Start Practice button | 64px height, 200px width minimum | `min-h-[64px] min-w-[200px]` |
| Dot progress indicators | display only — NOT interactive | No tap target; decorative |

Source: CLAUDE.md (44px minimum), Phase 1 D-12, `src/screens/HomeScreen.tsx` (64px CTA pattern)

### Tap Behavior

| Interaction | State Required | What Happens | Audio Effect |
|-------------|---------------|--------------|--------------|
| Tap anywhere on unlock screen | `unlock` | `howls[0].play()` synchronously, then `dispatch(UNLOCK)` | Step 0 audio starts |
| Audio finishes playing | `playing` | `dispatch(AUDIO_ENDED)` via Howl `onend` callback | — |
| Tap Next indicator | `between-steps` | `dispatch(ADVANCE)` | Next step audio starts (or celebration — no audio) |
| Tap replay icon | `playing` or `between-steps` | `howl.stop(); howl.play(); dispatch(REPLAY)` | Current step audio restarts from 0 |
| Tap Start Practice | `celebration` | `navigate('/practice')` | — |

### State Transition Visual Changes

| State | Visible UI Changes |
|-------|--------------------|
| `unlock` → `playing` (step 0) | Unlock screen fades out; StepCard slides in from right |
| `playing` → `between-steps` | Next indicator fades in (was hidden); replay icon remains visible |
| `between-steps` → `playing` (next step) | StepCard content replaces previous step; Next indicator hides; dot advances |
| `between-steps` → `celebration` | StepCard slides out; ConfettiScreen fades in |

### Step Transition Animation

- StepCard entrance: `opacity: 0 → 1` + `translateX(24px) → 0` over 200ms `ease-out`
- Implementation: CSS transition on the card div (not Motion/framer-motion — keep lightweight)
- Next indicator appearance: `opacity: 0 → 1` over 150ms `ease-in`
- No exit animation for StepCard — new step replaces immediately (avoids double-animation complexity)

Source: 03-CONTEXT.md Claude's Discretion ("animation for step transitions"), 03-RESEARCH.md Standard Stack (Motion is optional, only if spring physics needed — CSS sufficient here)

### Progress Indicator

- Dot count: one dot per LessonStep in `workedExample.steps[]`
- Active dot: `#FF6B35` filled circle, 12px diameter
- Inactive dot: `#FF6B35` at 20% opacity outline, 12px diameter
- Gap between dots: 8px
- Position: top-center of step screen, 24px from top safe-area edge
- Format: dots only, no "Step N of M" label (avoids clutter per D-07 discretion)

## Audio Behavior Contract

This section exists because audio drive state and visual transitions in this phase — UI and audio are tightly coupled.

| State | Audio Behavior | Visual Response |
|-------|---------------|-----------------|
| Unlock tap | `howls[0].play()` called synchronously in event handler | Screen transitions to step 0 |
| Step entered (`playing`) | `howls[stepIndex].play()` called — must be synchronous in dispatch side-effect | Audio plays; Next indicator hidden |
| Audio ends (`onend` fires) | `dispatch(AUDIO_ENDED)` | Next indicator fades in |
| Audio fails (404, `onloaderror`) | `dispatch(AUDIO_ENDED)` — CRITICAL: prevents stuck state | Next indicator fades in immediately |
| Replay tapped | `howls[stepIndex].stop(); howls[stepIndex].play()` | State → `playing`; Next indicator hidden |
| Celebration | No audio (Phase 3 scope only) | Confetti runs; no audio |

Source: 03-RESEARCH.md Pitfall 5 (onloaderror must dispatch AUDIO_ENDED), Pattern 1 (synchronous unlock)

## Accessibility Contract

| Element | Requirement | Implementation |
|---------|-------------|----------------|
| Unlock screen | Readable by screen reader | `<h1>` for lesson title; instruction text in `<p>` |
| RemyFox SVG | Already has `aria-label="Remy the Fox" role="img"` | No change needed |
| Replay icon | Must be keyboard/assistive reachable | `<button>` element, `aria-label="Replay this step"` |
| Next indicator | Must be keyboard/assistive reachable | `<button>` element, `aria-label="Next step"` |
| Step text | No action needed | Rendered in `<p>` — screen readers read it naturally |
| Equation | Read as text | Unicode equation strings are readable as-is (e.g., "3 + 4 = ?") |
| Progress dots | Context for screen readers | `aria-label="Step {N} of {M}"` on the dots container |
| Confetti | Decorative | `aria-hidden="true"` on confetti container |
| "You did it!" | Must be announced | `<h1>` on celebration screen |
