# Phase 1 — UI Design Contract

> Visual and interaction contract for the app shell, home screen, and stub routes.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> **Audience note:** All child-facing UI targets ages 6–9. Every visual decision
> (scale, color, copy) is calibrated for early readers on a 10-inch iPad display.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — hand-built Tailwind utility components |
| Icon library | none in Phase 1 (SVG inline where needed) |
| Font | Nunito (Google Fonts CDN) — all weights 400/600/700/800 |

**Rationale (from D-11):** Nunito is rounded, highly legible for early readers, and
covers the full weight range needed without a secondary typeface. No shadcn because
the component surface for Phase 1 is small (one real screen + three stubs) and the
kids-app aesthetic requires non-default visual styling that shadcn presets would
fight against.

---

## Spacing Scale

All values are multiples of 4. Applied via Tailwind v4 utility classes (`gap-2` = 8px, etc.)
— no custom spacing tokens needed beyond the default Tailwind scale.

| Token | Value | Primary Usage |
|-------|-------|---------------|
| xs | 4px | Icon-to-label gaps, tight inline spacing |
| sm | 8px | Button internal padding (vertical), label grouping |
| md | 16px | Horizontal screen edge padding, between-element gaps |
| lg | 24px | Section vertical rhythm, mascot-to-heading gap |
| xl | 32px | Heading-to-CTA gap |
| 2xl | 48px | Mascot top clearance from viewport top |
| 3xl | 64px | Not used in Phase 1 |

**Exceptions:**
- Primary CTA button: minimum 64px height (oversized touch target for age group — exceeds 44px Apple HIG minimum intentionally)
- Parent entry button: minimum 44×44px (meets Apple HIG exactly — unobtrusive)
- Safe-area bottom inset: `env(safe-area-inset-bottom)` applied as padding-bottom on all screens — not a spacing token, applied in CSS reset

---

## Typography

Font family (all roles): `"Nunito", ui-sans-serif, system-ui, sans-serif`
Applied globally via `--font-sans` in `@theme {}` and `font-family: var(--font-sans)` on `html`.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (Regular) | 1.5 | Stub screen placeholder text, parent entry label |
| Label | 20px | 600 (SemiBold) | 1.4 | Home screen heading tagline, stub screen status text |
| Heading | 32px | 700 (Bold) | 1.2 | Home screen primary heading ("Math Time!") |
| Display | 24px | 700 (Bold) | 1.2 | Primary CTA button label ("Start Learning") |

**Rules:**
- Maximum 4 sizes in use per screen — home screen uses Heading + Display + Body only.
- Do not introduce italic in Phase 1. Italic weight (loaded for future lesson narration cues) is available but unused.
- All text on child-facing screens: `user-select: none` (CLAUDE.md, D-12).
- Parent entry button: Body size (16px), weight 400, color `--color-on-surface` at 40% opacity.

## Color

All colors defined in `@theme {}` block in `src/index.css`. Source: RESEARCH.md Pattern 2.
No `tailwind.config.js` — Tailwind v4 CSS-first configuration.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--color-surface` | `#FFF8F0` | App background — every screen's base fill |
| Secondary (30%) | `--color-on-surface` | `#1C1917` | All body text, labels, secondary button labels |
| Accent (10%) | `--color-primary` | `#FF6B35` | Primary CTA button background only |
| Supporting — Fox warm | `--color-secondary` | `#FFD23F` | Remy SVG ear highlights, inner face detail |
| Supporting — Blue | `--color-accent` | `#3B82F6` | Reserved for Phase 4 correct-answer feedback (not used in Phase 1) |
| Semantic — Success | `--color-success` | `#22C55E` | Reserved for Phase 4 correct feedback (not used in Phase 1) |

**Accent reserved for (Phase 1 specifically):**
- Primary CTA button background (`bg-primary`) — the single "Start Learning" button only.
- Remy SVG fox body fill — the mascot illustration.

**60/30/10 breakdown:**
- 60%: `#FFF8F0` warm off-white fills every screen background and all stub screen content areas.
- 30%: `#1C1917` near-black used for all text content.
- 10%: `#FF6B35` orange appears on the one CTA button and the fox illustration body. Together these two elements occupy roughly 10% of visual real estate on the home screen.

**Color-scheme:** Light only. Add `<meta name="color-scheme" content="light">` in `index.html` to prevent Safari dark mode inversion.

## Screen Layouts

### Home Screen (`/`)

```
┌────────────────────────────────────────┐
│  [safe-area-inset-top]                 │
│                                        │
│         [Remy the Fox SVG]             │  ← 192×192px (w-48 h-48)
│         centered, ~48px from top       │
│                                        │
│         "Math Time!"                   │  ← Heading 32px/700, centered
│                                        │  ← 24px gap below Remy
│  ┌──────────────────────────────────┐  │
│  │        Start Learning            │  │  ← Display 24px/700, white text
│  │   (orange, 64px min-height)      │  │  ← bg-primary, rounded-3xl, full-width-ish
│  └──────────────────────────────────┘  │
│                                        │
│              Parent                    │  ← Body 16px/400, on-surface/40%, underline
│                                        │
│  [safe-area-inset-bottom]              │
└────────────────────────────────────────┘
```

**Layout system:**
- Container: `flex flex-col items-center justify-center`, `height: 100dvh`
- Padding: `px-6` (24px horizontal), `paddingBottom: env(safe-area-inset-bottom)` inline style
- Gap between mascot, heading, CTA: `gap-8` (32px)
- Gap between CTA and parent button: `gap-4` (16px) or positioned with `mt-auto` if layout needs breathing room

**Mascot (Remy the Fox):**
- Rendered as `<RemyFox className="w-48 h-48" />` (192×192px)
- Inline SVG, no external asset fetch
- `role="img"` and `aria-label="Remy the Fox"` for accessibility
- No animation in Phase 1 — static SVG (D-06, deferred per CONTEXT.md deferred section)
- Colors: body `#FF6B35`, ear highlight/face `#FFD23F`, eyes/nose `#1C1917`, eye shine `white`

**Primary CTA — "Start Learning":**
- `min-height: 64px` (intentionally exceeds 44px — large, inviting target for young children)
- `min-width: 200px`, padding `px-10 py-5`
- Background: `bg-primary` (`#FF6B35`)
- Text: `text-white`, `text-2xl` (24px), `font-bold` (700)
- Border radius: `rounded-3xl` (very rounded — warm, approachable for kids)
- Active state: `active:opacity-80` — brief opacity dip on tap, no transform (avoids jank on older iPads)
- `touch-action: manipulation` (inline style or global CSS rule)
- No hover state (touch-first, D-01)

**Parent entry button:**
- Label: "Parent"
- `text-sm` (14px), `font-normal` (400), `text-on-surface/40` (40% opacity near-black), `underline`
- `min-height: 44px`, `min-width: 44px` (meets Apple HIG exactly)
- Active state: `active:opacity-60`
- `touch-action: manipulation`
- Positioned below CTA — visually de-emphasized, discoverable but not prominent (D-03)

## Touch Interaction Patterns

**Global rules (applied in CSS reset):**

```css
button, [role="button"], a {
  touch-action: manipulation;  /* eliminates 300ms tap delay */
  user-select: none;
  min-height: 44px;
  min-width: 44px;
}

* {
  -webkit-tap-highlight-color: transparent;  /* removes iOS blue flash */
}
```

**Per-element tap state:**

| Element | Resting | Pressed/Active | Disabled |
|---------|---------|---------------|----------|
| "Start Learning" CTA | `#FF6B35` background, white text | `opacity: 0.8` | Not applicable (always enabled) |
| "Parent" entry | `#1C1917` at 40% opacity, underline | `opacity: 0.6` | Not applicable |
| Stub lesson card | Static (not tappable) | N/A | N/A |
| Placeholder answer tiles | `bg-white` border | N/A (no handler) | N/A |

**No hover states** on any element. Touch-first; hover interactions are absent by design (D-01, CLAUDE.md).

**No focus rings** for child flow. Parent screen stubs may add focus-visible rings when PIN input is built in Phase 5.

**Touch target enforcement:** All tappable elements must have computed min-height and min-width of 44px as verified by `getBoundingClientRect()` in tests (PLAT-01 test).

## Animation and Transitions

**Phase 1 scope:** Minimal. No route transition animations — instantaneous screen swaps via React Router navigate().

**Rationale:** Motion library (Motion/Framer Motion) is not installed in Phase 1. Route transitions are a Phase 3+ concern when audio and lesson sequencing are established. Adding animation infrastructure now without audio sync would need to be redone.

**What IS animated in Phase 1:**
- None. The `active:opacity-*` CSS tap states are CSS transitions (not JS animation), applied natively by the browser — no library required.

**What is NOT animated (deferred):**
- Remy the Fox idle/wave/celebrate sequences — Phase 3 (Lottie)
- Route enter/exit transitions — Phase 3
- Correct answer celebration — Phase 4

## Viewport and Layout System

**Full-screen shell:**

```css
html, body, #root {
  height: 100dvh;
  width: 100%;
  margin: 0;
  padding: 0;
  overflow: hidden;
  font-family: var(--font-sans);
  background-color: var(--color-surface);
}

body {
  padding-bottom: env(safe-area-inset-bottom);
}
```

**Screen container pattern (applied to every screen component):**

```tsx
<div
  className="w-full flex flex-col items-center justify-center bg-surface text-on-surface"
  style={{
    height: '100dvh',
    paddingTop: 'env(safe-area-inset-top)',
    paddingBottom: 'env(safe-area-inset-bottom)',
  }}
>
  {/* screen content */}
</div>
```

**Orientation support:**
- Portrait (768×1024): Default target. Mascot centered vertically with `justify-center`.
- Landscape (1024×768): Same flex-col layout still works — mascot shrinks proportionally because it is sized in `w-48 h-48` (viewport-relative-safe), not fixed pixels. No layout rewrite needed.
- Do not use fixed pixel heights for any content region. All heights are `h-full`, `100dvh`, or `min-h-[Npx]`.

**Meta viewport tag:**

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

`viewport-fit=cover` is required — without it, `env(safe-area-inset-bottom)` evaluates to 0 (RESEARCH.md Pitfall 2).

**Color scheme meta (required):**

```html
<meta name="color-scheme" content="light" />
```

Prevents Safari automatic dark mode inversion of the warm palette.

**Tone rules (child-facing screens):**
- Warm, encouraging, present tense where possible.
- No negatives ("No lessons yet") — use forward-looking framing ("Coming soon").
- Short sentences. Ages 6–9 read at roughly a grade 1–3 level.

**Tone rules (stub screens):**
- Stubs are developer-facing context, not child-facing. But they still use the app's visual design — the copy just clarifies the stub status for developers reviewing the build.

**Destructive actions in Phase 1:** None. No delete, reset, or confirmation flows exist.

## CSS Token Block Reference

The complete `@theme {}` block for `src/index.css` (from RESEARCH.md Pattern 2):

```css
@import url('https://fonts.googleapis.com/css2?family=Nunito:ital,wght@0,400;0,600;0,700;0,800;1,400&display=swap');

@import "tailwindcss";

@theme {
  --font-sans: "Nunito", ui-sans-serif, system-ui, sans-serif;

  --color-primary:    #FF6B35;   /* warm orange — CTA button, fox body */
  --color-secondary:  #FFD23F;   /* bright yellow — fox ears, reserved warm accent */
  --color-accent:     #3B82F6;   /* vivid blue — Phase 4+ correct feedback */
  --color-success:    #22C55E;   /* green — Phase 4+ correct answer */
  --color-surface:    #FFF8F0;   /* warm off-white — universal background */
  --color-on-surface: #1C1917;   /* near-black — all text */
}

html, body, #root {
  height: 100dvh;
  width: 100%;
  margin: 0;
  padding: 0;
  overflow: hidden;
  font-family: var(--font-sans);
  background-color: var(--color-surface);
}

body {
  padding-bottom: env(safe-area-inset-bottom);
}

* {
  -webkit-tap-highlight-color: transparent;
}

button, [role="button"], a {
  touch-action: manipulation;
  user-select: none;
  min-height: 44px;
  min-width: 44px;
}
```

**Critical ordering:** `@import url(...)` for Google Fonts MUST appear before `@import "tailwindcss"`. Reversed order causes font import hoisting conflict.

## Accessibility Baseline

| Requirement | Implementation |
|-------------|----------------|
| Touch target 44×44px minimum | All buttons: `min-height: 44px; min-width: 44px` in global CSS |
| Mascot image alternative | `<RemyFox role="img" aria-label="Remy the Fox" />` |
| Color contrast (body text) | `#1C1917` on `#FFF8F0` — contrast ratio ~18:1 (WCAG AAA) |
| Color contrast (CTA button) | White on `#FF6B35` — contrast ratio ~3.4:1 (WCAG AA for large text — 24px bold qualifies) |
| Color contrast (parent link) | `#1C1917` at 40% opacity on `#FFF8F0` — ~5.2:1 (WCAG AA) |
| Tap highlight | Suppressed globally (`-webkit-tap-highlight-color: transparent`) |
| Text scaling | Nunito at defined px sizes — not using `rem`-based sizing in Phase 1; acceptable for controlled iPad viewport |
